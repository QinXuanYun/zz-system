# zz-system 系统修复记录 - 2026-04-16

根据《系统开发修改意见0416.docx》文档，已逐条修复以下问题：

## 已修复问题清单

### 1. 蓝色指标判断逻辑修复 ✅
**问题**: 蓝色指标代表的是数据在正常范围内，且下降大于等于5%，但数据有上升时也显示为蓝色指标。

**修复内容**:
- 修改了 `backend/main.py` 中的 `get_level_value()` 函数
- 添加了 `prev_val` 参数用于计算与上一年的变化
- 蓝色指标现在只在以下情况显示：
  - 数据在正常范围内（>= green threshold）
  - 与上一年相比下降 >= 5%（对于越高越好的指标）
  - 或上升 >= 5%（对于越低越好的指标，如生师比）

### 2. 趋势分析图纵坐标间隔修复 ✅
**问题**: 趋势分析图纵坐标间隔为5，导致数据变化趋势不明显，且很多指标数据叠加在一起。

**修复内容**:
- 修改了 `frontend/index.html` 中的 `renderTrendChart()` 函数
- 根据数据范围动态计算纵坐标间隔：
  - 数据范围 <= 5：间隔为1
  - 数据范围 <= 10：间隔为2
  - 数据范围 <= 20：间隔为5
  - 数据范围 > 20：间隔为10
- 同时设置合理的 min/max 值使图表显示更清晰

### 3. 雷达图灰色线圈（平均值）修复 ✅
**问题**: 雷达图中灰色的线圈不代表指标数据平均值，每个指标的平均值不应该相同。

**修复内容**:
- 修复了三个地方的雷达图：
  1. 专业详情页单专业雷达图 (`renderSingleRadar`)
  2. 专业对比页雷达图 (`renderRankingCompareChart`)
  3. 报告页雷达图 (`renderReportRadar`)
- 灰色线圈现在正确显示每个指标在所有专业中的平均值
- 平均值线使用虚线样式，与专业数据线区分

### 4. 最高分/最低分统计逻辑修复 ✅
**问题**: 报告中每个专业的最高分/最低分数量统计不正确，未考虑指标方向性。

**修复内容**:
- 修改了 `backend/main.py` 报告生成部分的统计逻辑
- 对于越高越好的指标：最大值=最高分，最小值=最低分
- 对于越低越好的指标（如生师比）：最小值=最高分，最大值=最低分
- 这样排名靠前的专业会有更多最高分指标，符合常理

### 5. 专业对比数据计算修复 ✅
**问题**: 专业对比部分，生师比、教师人均企业实践时间、师均论文数等数据计算不正确。

**修复内容**:
- 修改了 `backend/main.py` 中的 `get_compare()` API
- 修复了各指标转换为0-100分值的计算逻辑：
  - 百分比指标：直接使用百分比值
  - 生师比：18以下为满分100，22以上为0分，中间线性插值
  - 天数指标：30天为满分100
  - 数值指标（如师均论文数）：1.5为满分100

### 6. 数据格式转换修复 ✅
**问题**: 导入时百分比数据的格式转换可能存在问题。

**修复内容**:
- 在 `get_compare()` API 中添加了额外的格式检查
- 确保百分比值大于1时正确转换为小数格式
